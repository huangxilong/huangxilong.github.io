从源码学习 WeUI 2

发表于 2017年05月05日 | 作者 黄乙 | 分类于 前端 |

上一篇文章【从源码学习 WeUI】主要内容是：学习 WeUI 源码目的，开发环境与工具，源码根目录的主要目录和文件简介，安装、运行 weui，gulp 的主要任务简单说明。本文接着继续深入学习。

源码目录有两个子目录，example 和 style。

  • example，从名称就可以知道，这个目录的代码是用来演示 WeUI 使用方法的。包括：按钮、列表、弹框、进度条、提示、文章、响应菜单、图标等各式组件。
  • style，存放着 WeUI 的基础样式库源码。我们的学习过程，就是要把这个目录的每个文件都读一遍或以上，理解并掌握。

example 目录

子目录与文件说明：

  • fragment 目录，各式独立组件的示例代码块。
  • images 目录，存放示例中需要用到的图片文件。
  • snapshot 目录，存放演示的实际屏幕快照图片。
  • example.js，主要功能有：预加载部分图片文件，优化手机端点击事件，管理各组件示例界面。
  • example.less，演示的样式表文件，使用了 Less 预处理语言。
  • index.html，网页基本结构文件。
  • zepto.min.js，为现代手机浏览器推出的 js 库。

这里有两点跟我们平常写的网页不一样。

  1. 各式组件放在 fragment 目录，分成了很多个代码块文件。
  2. example.less 这个文件跟常见的 css 文件不太一样。

style 目录

基础样式库源码目录与文件说明：

  • base 目录，常用的 mixin（手机定制、箭头、1像素、文本），基础变量（颜色、字体、组件参数），重置浏览器默认样式。
  • icon 目录，是用 IconFont 实现常用图标。
  • widget 目录，独立组件的源码。
  • weui.less，WeUI 主文件，就是导入了上面目录的文件。

可以看到，WeUI 项目是精心设计的，抽离了常用的 mixin、基础变量，对常用图标也单独处理，常用的组件也统一放到组件目录，再按照不同的组件放在独自的子目录里。这样子可以减轻后续维护的成本，同时也对学习这个产品的思路比较清晰。

Less 预处理语言

Less 是一种 CSS 扩展，使用现有的 CSS 语法新增了额外的特性，而且向后兼容 CSS。简单说，我们可以使用新的扩展语法，也可以使用标准的 CSS语法。

下面简要说明一下常用的扩展语法：

  • 导入(Importing)，引入外部文件。
  • 变量(Variables)，例如颜色、大小可以统一定义，要修改也只改一下就可以了。
  • 混合(Mixins)，可以将一系列属性从一个规则集引入到其它的规则集里。
  • 嵌套规则(Nested rules)，模仿了 HTML 的结构，把子元素定义在父元素里面。
  • 运算(Operations)，数值，颜色和变量都可以进行运算。
  • 函数(Functions)，Less 提供了颜色，处理字符串和进行算术运算函数。

详细的 Less 参考手册请到官网（lesscss.org）上查阅更全面。

有了以上的初步了解，以及 Less 的熟悉，我们后面可以逐个组件来学习 WeUI 的使用和源码了。
