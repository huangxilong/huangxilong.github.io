BEM 命名规则

发表于 2017年04月12日 | 作者 黄乙 | 分类于 前端 |

最近使用 weui 开发移动端网页。其中的 CSS 命名方式勾起了我的兴趣。特意去了解它的来由。原来一开始 weui 的命名规范是采用「模块+应用场景」的方式，没有对连字符和层级做严格区分，团队内约定使用的都是下划线。后来改用了 BEM 的命名规则，横杠是连词符，双下划线区分 block 与 element，单下划线后面跟 modifier。相比「模块+应用场景」，BEM 命名规则更利于代码的开发与维护，也广受开发者的接受。

BEM 是什么？

BEM 是 Yandex 团队提出的一种前端命名规则，分别是：块（block）、元素（element）、修饰符（modifier）的首字母。以 weui 为例：block 与 element 之间用双下划线区分，modifier 用单下划线区分。

宝剑锋从磨砺出，梅花香自苦寒来。

我们做前端开发时，经常会碰到一些琐碎、重复的小任务。像压缩、合并文件，语法检查，编译 Sass/LESS，更新文件后刷新浏览器，等等。虽然这些只是花一两分钟就能做的小事情，但是每当重复一次又一次这些工作的时候，心里总会有些不爽。这时 gulp 能够把你从不爽中拯救出来，在不知不觉中提高了工作效率。

gulp 是什么？

它是一个自动化构建工具，我们可以使用它在项目开发过程中自动执行常见任务。gulp 可以直接使用代码来配置和执行各种小任务，也可以通过现成的插件来完成我们定义的工作。

发表于 2017年03月08日 | 作者 黄乙 | 分类于 写作 |

你需要使用 Markdown 吗？

Markdown 是一种纯文本标记语言，目的是希望大家使用「易于阅读、易于撰写的纯文字格式」来写作。如果你想要写博客文章，或者做文字类的内容，可以了解一下 Markdown。用户只要使用一些简单的标记符号，就能生成极富表现力的文档。其特点如下：

  • 纯文本，可以使用任意文本编辑器写作。
  • 专注于内容，标记语法非常轻量且可读性强。
  • 内容与样式分离，且可以导出为 PDF 或 HTML 文件。
